# Maintenance Guide

Checklist of things to update periodically when using this template.

---

## Regular Updates

### Python version
File: `backend/.python-version` and `.devcontainer/Dockerfile`

```bash
# Check latest stable release
uv python list

# Update .python-version
echo "3.12.X" > backend/.python-version

# Update Dockerfile line
# uv python install 3.12.X
```

Also update the version string in `backend/pyproject.toml` (`requires-python`) and the CI workflow (`python-version`).

---

### Backend dependencies
File: `backend/pyproject.toml` + `backend/uv.lock`

```bash
cd backend
uv lock --upgrade          # Upgrade all deps within constraints
uv sync --all-extras       # Re-sync venv
```

Pin major versions in `pyproject.toml` for production stability.

---

### Frontend dependencies
File: `frontend/package.json` + `frontend/bun.lock`

```bash
cd frontend
bun update                 # Upgrade all deps
bun run check              # Verify no type errors after upgrade
```

---

### Pre-commit hooks
File: `.pre-commit-config.yaml`

```bash
pre-commit autoupdate      # Updates all hook revisions to latest tags
pre-commit run --all-files # Verify nothing broke
```

The ruff version here should stay in sync with the ruff version in `backend/pyproject.toml` dev deps.

---

### Node.js version
File: `.devcontainer/Dockerfile`

```dockerfile
# Change the nvm install target version
nvm install 24   # update to desired LTS
```

Also update `frontend/.npmrc` if you add `node>=X` engine constraints.

---

### GitHub Actions
File: `.github/workflows/ci.yaml`

Pin action versions (`actions/checkout@v4`, `astral-sh/setup-uv@v5`, `oven-sh/setup-bun@v2`) to the latest stable major version periodically.

Use Dependabot or Renovate to automate this:

```yaml
# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"
```

---

## When Using This Template

Replace all `{{ placeholder }}` tokens before first commit:

| Placeholder | Location | Replace with |
|---|---|---|
| `{{ project_name }}` | `README.md` | Your project name |
| `{{ author_name }}` | `backend/pyproject.toml` | Your name |
| `author@example.com` | `backend/pyproject.toml` | Your actual email |

Also:
- Copy `.env.example` to `.env` and fill in real values
- Change `name` in `frontend/package.json` from `frontend` to your project name
- Update `name` in `backend/pyproject.toml` from `backend` to your project name
- Remove or replace the `LICENSE` file if Apache 2.0 is not appropriate
